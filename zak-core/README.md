# ZAK: Zero-Asset Kernel (v1.0.1 GOLD)

ZAK is a security-critical, deterministic AI execution kernel designed for high-stakes enterprise "repatriation" (moving AI workloads from Cloud to local infrastructure).

## Core Architecture

- **Validator**: SHA-256 integrity, geometry enforcement, resonance safety bands.
- **Compiler**: Deterministic manifest generation, material physics mapping.
- **Runner**: Hard-sliced word limits, JSON repair, zero-memory execution.
- **Tuner**: Drift-proof Sigil evolution with immutable write contracts.
- **Adapters**: Hardware-agnostic bridges for `llama.cpp` and `vLLM`.

## Project Structure

```text
src/kernel/      # The Security-Critical Core
src/adapters/    # Hardware Bridge Layer
src/ui/          # Minimal Pilot UI
zak-zem/         # Enterprise Management Console
scripts/         # Verification & Pilot Bootstrap
tests/           # Red-Team, Determinism, and TCO proofs
sigils/          # Sealed Governance Contracts
```

## Getting Started

### 1. Final Verification
Prove the system integrity on your local hardware:
```bash
npm run verify
```

### 2. Pilot UI
Launch the minimal, zero-state user terminal:
```bash
npm run ui
```

### 3. Enterprise Manager (ZEM)
Launch the full hardware telemetry and governance console:
```bash
# Windows
zak-zem/start.bat
# Linux/Mac
zak-zem/start.sh
```

## Security Invariants

1. **Zero Drift**: 1,000 identical runs produce 1,000 identical manifests.
2. **Zero Memory**: No user data persists across execution boundaries.
3. **Mechanical Safety**: Attacks are neutralized by the compiler, not the model.
4. **Air-Gap Ready**: No telemetry, no phone-home, no external dependencies.
